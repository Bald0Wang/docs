## 本次检查点目标

将 Demo 进化为真正“好用、稳定、可解释”的产品：不仅能跑，还能被用户用起来、被评委验收、被队友复现。

> 关键词：数据驱动、敏捷迭代、评测回归、工程化、可信体验

## 1. 通识必修 (全员)

- **敏捷进阶**：学习 Scrum 流程与 Sprint Planning。
- **数据思维**：设定并追踪“北极星指标”。
- **评测意识**：建立团队内部的 AI 质量评估标准。
- **AI 协作流程化**：Prompt Library + 失败案例库 + 输出质检（反幻觉/反伪引用）。

### Sprint 节奏建议（每周一个小循环）
- **Planning（计划）**：本周要交付的“可演示增量”是什么？
- **Daily（站会）**：阻塞与风险透明化
- **Review（评审）**：用演示说话（不要只讲计划）
- **Retro（复盘）**：把坑写进规则/模板里，避免重复踩

### 北极星指标（示例）
- 效率型：完成一次任务平均节省时间（分钟）
- 质量型：任务成功率 / 用户满意度（1–5）
- 可信型：引用可追溯比例 / 幻觉命中率（越低越好）

## 2. 专精技能树（分角色任务书）

> 说明：这一阶段的核心是“建立闭环”：有验证、有数据、有迭代、有质量保障。

### 市场方向

#### 核心任务
- **定量调研**：问卷设计基础（避免诱导性提问）；最少样本也要说明偏差与限制。
- **商业模式**：BMC 拆解（成本结构、收入流、渠道通路、关键资源）。
- **增长实验**：冷启动策略与种子用户获取；用实验日志记录“假设→实验→指标→结论”。
- **证据链升级**：把“观点”变成“证据 + 结论 + 风险”三段式表达。

#### 增长实验日志（模板）

| 字段 | 说明 |
| :--- | :--- |
| 假设 | 例如：校园社群能带来 20 个种子用户 |
| 实验动作 | 发什么内容、在哪发、怎么引导 |
| 指标 | 曝光/点击/转化/留存/有效反馈数 |
| 结果 | 数据 + 关键截图/链接 |
| 结论 | 是否成立；下一步怎么改 |

### 产品方向

#### 核心任务
- **设计系统（轻量版）**：色彩/字体/组件命名/状态规范（至少覆盖核心页面）。
- **交互逻辑进阶**：异常状态（断网/超时/无权限/AI 拒答/AI 瞎编）必须可用且可解释。
- **可信体验**：引用来源展示、置信度提示、可撤销与可追溯（最少实现一种）。
- **可用性测试**：设计测试脚本（任务、成功标准、记录方式），用数据推动改版。
- **A/B 试验（小规模）**：两套方案至少做一次盲测或对照体验。

#### DRD（高保真交互文档）建议目录
- 目标与范围（冻结范围引用自 Milestone 1 输出）
- 核心页面与流程（主链路）
- 状态机（关键组件/关键页面）
- 异常流与空状态（必须齐）
- AI 交互规范（输入、输出格式、拒答、引用与边界）
- 埋点与指标（对应北极星指标）
- 验收标准（可被市场同学 UAT 验收）

### 技术方向

#### 核心任务
- **端到端闭环优先**：先跑通一条最小链路，再做体验与性能优化。
- **应用进阶**：引入 RAG（检索增强生成）降低幻觉风险（如项目需要知识型问答/文档检索）。
- **系统架构**：数据存储、权限边界、异步任务（耗时任务不阻塞体验）。
- **可观测性**：日志/关键指标/错误追踪（能定位“为什么回答变差”）。
- **部署运维**：容器化与云端部署；让 Demo “别人也能用”，不是“只能在我电脑上跑”。
- **评测与回归**：建立用例集（成功路径 + 失败案例），每次改动必须回归。

#### 最小评测用例表（模板）

| 用例ID | 场景 | 输入 | 期望输出 | 失败判定 | 备注 |
| :--- | :--- | :--- | :--- | :--- | :--- |
| TC-001 | 主链路 | … | … | … | … |
| TC-002 | 幻觉风险 | … | 必须引用来源/必须拒答 | 编造引用 | … |
| TC-003 | 异常流 | … | 友好错误与引导 | 卡死/无提示 | … |

## 3. Milestone 2：中期评审（Sprint Review）

### 评审输入
- 可演示的 MVP（主链路必须跑通）
- DRD v1（含异常流/空状态/AI 交互规范）
- 最小评测用例表 + 回归记录（至少覆盖主链路与 1 个风险场景）
- 商业验证与增长实验日志（至少 1 次实验 + 结论）

### 评审输出
- 是否进入“打磨期”的准入结论（通过/需整改）
- 需求范围是否需要二次冻结（只做减法，不做加法）
- 下一步打磨清单（稳定性/体验/表达/合规）
