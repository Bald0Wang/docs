## 1. 角色定位（你要把“想法”变成“可被实现与验收的体验”）

产品角色是团队的“体验架构师”：把市场的洞察与证据，翻译成**清晰的主链路、页面结构、状态机与 AI 交互规范**，让技术能实现、市场能验收、评委能看懂。

### 你在团队里的典型交付物

- **Milestone 1（需求评审）**：《核心流程低保真线框图》+ 关键状态清单 + AI 交互说明 v0
- **Milestone 2（中期评审）**：《高保真交互文档（DRD v1）》+ 可用性测试记录 + 指标/埋点口径
- **Milestone 3（最终验收）**：作品集 Case Study + 演示视频/走查清单 + Pixel 级验收结论

> 对应检查点：[`实战里程碑`](/hackathon/milestones)。

## 2. 段位成长路线（初级 / 中级 / 高级）

> 使用方式：直接对照你目前的段位，把“要学什么 / 要掌握什么 / 能交付什么”补齐；阶段页与里程碑只作为进度检查点（见第 1 节链接）。

### 初级：能把需求变成“可实现的主链路”，并补齐失败路径

- **需要学习什么（方法）**
  - 需求做减法：只保 1 条主链路，其他先进入“不做清单”
  - 低保真表达：让队友一眼看懂“点哪里、看到什么、下一步是什么”
  - 状态意识：空/错/加载/超时/无权限/AI 拒答与不确定
  - AI 交互基础：AI 的角色、输入来源、输出格式、失败怎么处理
- **需要掌握什么（能力标准）**
  - 能画出“从触发到价值”的主流程（不追求精致，追求可落地）
  - 至少补齐 **空状态 / 错误状态 / AI 不确定或拒答状态**
  - 能把“成功长什么样、失败怎么提示”说清楚，方便技术实现与市场验收
- **你应该能交付什么（最小交付包）**
  - 核心流程低保真线框图（只覆盖主链路）
  - 关键状态清单（含 AI 失败体验）
  - AI 交互说明 v0（输入/输出/禁止事项/失败处理）
  - 演示脚本草案（主链路 1 次 + 失败处理 1 次）

### 中级：能用 DRD 把实现、验收与迭代“标准化”

- **需要学习什么（方法）**
  - DRD 写法：范围、流程、状态机、异常流、AI 规范、验收标准
  - 可用性测试：测试脚本、记录方式、结论与改版
  - 指标与埋点口径：北极星指标如何被观察（先手工也可以）
  - 小规模对照：A/B 或盲测（两套交互/两套文案）
- **需要掌握什么（能力标准）**
  - 原型/DRD 能直接指导开发（减少口头解释与反复返工）
  - 验收标准可执行：市场同学能按清单做 UAT
  - 迭代有依据：能把用户反馈转成“可复现问题清单 + 优先级”
- **你应该能交付什么（稳定交付包）**
  - DRD v1（覆盖主链路 + 必要异常流）
  - 可用性测试记录（至少 1 轮）+ 结论与改版点
  - 体验走查清单（卡住点/误解点/多余输入）
  - UAT 验收点清单（按流程逐条可判定）

### 高级：能带队建立体验体系（设计系统 / 状态机 / 可信体验 / 表达）

- **需要学习什么（方法）**
  - 轻量设计系统：色彩、字体、组件命名、状态规范
  - 复杂交互治理：关键组件状态机、回退与恢复、错误归因提示
  - 可信体验：引用来源/置信提示/可撤销/可追溯（至少落地一种）
  - 作品集表达：Case Study 的结构化叙事与权衡呈现
- **需要掌握什么（能力标准）**
  - 能组织每周一个可演示增量，让“文档—实现—验收”同步
  - 能统一团队对外呈现：演示顺序、文案口径、失败处理展示方式
  - 能做 Pixel 级体验验收（尤其是主链路关键页与组件一致性）
- **你应该能交付什么（带队交付包）**
  - 轻量设计系统（覆盖核心页面/组件/状态）
  - 关键页面/组件状态机说明 + 可信体验方案（已落地）
  - 作品集 Case Study（含证据与权衡）+ 演示视频
  - Pixel Perfect 验收结论（走查记录 + 整改清单）

## 3. 互锁协作：你必须卡住的三件事

- **与市场（Market）**：证据不充分就不扩流程；验收标准与指标口径必须一致
- **与技术（Tech）**：原型与 DRD 是开发依据；原型变更必须同步到看板与 DoD
- **对团队（All）**：必须展示“失败处理”（AI 不确定/拒答/错误）才算可用

## 4. 技能树（Product Skill Tree）

- **问题到体验的转化**
  - 需求拆解：主链路 / 次链路 / 不做清单
  - 信息架构（IA）：页面结构、导航、内容层级
  - 流程设计：触发→输入→处理→输出→价值闭环
- **交互与状态机**
  - 空状态 / 错误状态 / 加载与超时 / 无权限
  - 关键组件状态机（状态、触发条件、回退与恢复）
  - 可访问性与可理解性（提示、引导、容错）
- **AI 交互设计**
  - AI 角色定义（生成/检索/判断/总结）
  - 输入/输出格式规范（结构化输出、引用展示）
  - 失败体验（拒答、澄清问题、边界与免责声明）
  - 可信体验（引用/置信提示/可追溯，至少落地一种）
- **验证与迭代**
  - 可用性测试：脚本、记录、结论与改版
  - A/B 或盲测（小样本也要写清限制与偏差）
  - 指标与埋点口径（北极星指标可被观察）
- **交付与表达**
  - 低保真线框图 / DRD（高保真交互文档）
  - 演示脚本与视频
  - 作品集 Case Study（过程、权衡、证据、结果）
