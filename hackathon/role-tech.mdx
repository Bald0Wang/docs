## 1. 角色定位（你要把“原型”变成“可稳定交付的系统”）

技术角色是团队的“引擎 + 稳定官”：把产品原型落地为可运行的端到端闭环，并用工程化手段保证**可复现、可评测、可部署、可解释**。

### 你在团队里的典型交付物

- **装备期**：技术可行性评估 v0（链路/依赖/成本/风险/备选/DoD）
- **深耕期**：稳定可演示的云端服务 + 最小评测用例表 + 回归记录
- **蜕变期**：开源级 README/API 说明 + 可复现部署 + 重构与稳定性报告（或技术博客）

> 对应阶段与里程碑：[`装备期`](/hackathon/stage-1-prep)、[`深耕期`](/hackathon/stage-2-deep-dive)、[`蜕变期`](/hackathon/stage-3-transformation)、[`实战里程碑`](/hackathon/milestones)。

## 2. 段位成长路线（初级 / 中级 / 高级）

> 使用方式：直接对照你目前的段位，把“要学什么 / 要掌握什么 / 能交付什么”补齐；阶段页与里程碑只作为进度检查点（见第 1 节链接）。

### 初级：能跑通端到端闭环，并把边界与风险说清楚

- **需要学习什么（方法）**
  - 端到端优先：先跑通一条最小链路，再谈架构与优化
  - 依赖梳理：模型/接口/数据/第三方服务的主备方案
  - 基础鲁棒性：超时、异常输入、错误提示与兜底策略
  - 成本与边界：预算上限、能力边界、合规红线
- **需要掌握什么（能力标准）**
  - 能跑通“输入→处理→输出→可演示价值”的最小链路（可复现、可解释）
  - 能明确：关键依赖是什么、最大风险是什么、做不到怎么降级
  - 不承诺做不到的事：能把需求砍成赛程内可交付闭环
- **你应该能交付什么（最小交付包）**
  - 技术可行性评估 v0（链路/依赖/成本/风险/备选/DoD）
  - 可演示的最小原型（哪怕是最简 UI/接口）
  - 最小运行说明（环境/启动/示例输入/预期输出）

### 中级：能把系统做“稳定、可复现、可回归”

- **需要学习什么（方法）**
  - 工程协作：分支/PR/Review/回滚（避免口头合并）
  - 评测与回归：用例集（主链路 + 风险场景）+ 回归记录
  - 可观测性：关键日志、错误追踪、核心指标（耗时/成功率/拒答率）
  - 部署与演示环境固定：让“别人也能用/能验收”
- **需要掌握什么（能力标准）**
  - 能稳定演示：异常不崩、超时可控、失败有提示
  - 每次改动都有回归：主链路不被自己改坏
  - 能把产品原型翻译成可执行 DoD（接口/字段/状态/错误口径）
- **你应该能交付什么（稳定交付包）**
  - 稳定可用的演示版本（最好可云端访问或可复现运行）
  - 最小评测用例表 + 回归记录（至少覆盖主链路与 1 个风险场景）
  - 最小可观测性说明（记录什么、怎么看、怎么定位问题）

### 高级：能带队做选型、架构与交付标准化（不翻车）

- **需要学习什么（方法）**
  - 选型与权衡：效果/成本/延迟/可维护/可扩展
  - 架构设计：数据存储、权限边界、异步任务（耗时任务不阻塞）
  - 稳定性治理：限流、重试、缓存、成本控制与降级策略
  - 开源级交付：README、架构说明、合规与许可、复现与贡献规范
- **需要掌握什么（能力标准）**
  - 能给出明确选型结论，并解释“为什么不选另一个”
  - 风险可验收：隐私/数据来源/引用可追溯/拒答策略写成条款并可验证
  - 现场可控：出问题能快速定位与降级，保证演示不翻车
- **你应该能交付什么（带队交付包）**
  - 架构说明（数据流/组件职责/关键依赖/替代方案）
  - 可复现部署交付包（云端或一键运行）+ 稳定性治理说明
  - 开源级 README/API 说明 + 合规与许可章节
  - 技术复盘/博客（关键难点、权衡、降级策略、结论）

## 3. 互锁协作：你必须卡住的三件事

- **与产品（Product）**：无原型/无 DRD 不开工；原型变更要同步到实现与回归用例
- **与市场（Market）**：无验收条款不算完成；风险与边界必须写进交付包与路演口径
- **对团队（All）**：端到端闭环优先于“某个模块做得很炫”

## 4. 技能树（Tech Skill Tree）

- **工程基础**
  - Git 工作流（分支/PR/Review/回滚）
  - 依赖与环境（版本锁定、配置管理、可复现运行）
  - 错误处理与鲁棒性（超时、重试、降级、兜底提示）
- **AI 应用能力**
  - 模型/接口选型（效果、成本、延迟、可维护）
  - Prompt 与结构化输出（可控、可解释）
  - RAG/检索增强（需要知识型问答/文档检索时）
  - 反幻觉策略（引用、拒答、澄清问题）
- **评测与质量**
  - 用例集设计（主链路 + 风险场景）
  - 回归记录（每次迭代都可验证）
  - 可观测性（日志、指标、错误追踪）
- **部署与交付**
  - 云端部署与可访问 Demo（别人能用）
  - 成本控制与稳定性治理（限流、缓存、异步任务）
  - 文档与开源规范（README、架构说明、合规与许可）
