## 1. 角色定位（你要回答的三句话）

- **为什么做**：这是不是“真痛点”？证据是什么？
- **给谁做**：目标用户是谁？在哪个场景里会愿意换用你的方案？
- **做到什么算成**：成功指标是什么？怎么验收？如何闭环？

你是团队的“雷达 + 验收官”：保证团队做的是**正确且值得做的事**，并在最后能用证据说服评委/企业/用户。

### 你在团队里的典型交付物

- **Milestone 1（需求评审）**：问题定义一页纸（Problem Statement）+ 竞品/替代方案矩阵 + 证据链
- **Milestone 2（中期评审）**：增长实验日志 + 商业验证报告（含指标与偏差说明）
- **Milestone 3（最终验收）**：路演叙事（Pitch Deck + 讲稿 + Q&A）+ UAT 验收结论 + 行业洞察/白皮书（可选加强）

> 对应检查点：[`实战里程碑`](/hackathon/milestones)。

## 2. 段位成长路线（初级 / 中级 / 高级）

## 2. 段位成长路线（初级 / 中级 / 高级）

> 使用方式：直接对照你目前的段位，把“要学什么 / 要掌握什么 / 能交付什么”补齐；阶段页与里程碑只作为进度检查点（见第 1 节链接）。

### 初级：能交付“证据 + 结论”，不拖团队后腿

- **需要学习什么（方法）**
  - 访谈与调研基础：如何问、如何追问、如何记录、如何提炼可引用原话
  - 竞品/替代方案分析：看“能力差异”而不是列名单
  - 问题定义：把“情绪化描述”写成“可观察行为 + 场景约束”
- **需要掌握什么（能力标准）**
  - 任何结论都能给出证据来源（访谈摘录/公开数据/真实案例）
  - 能说清：目标用户是谁、在什么场景、为什么痛、现在怎么解决
  - 能识别并写出反例（不只收集支持自己观点的样本）
- **你应该能交付什么（最小交付包）**
  - 《问题定义一页纸》（含 UVP/指标/边界）
  - 最小访谈纪要（建议 5 人也可）：**3 条可引用原话 + 2 条反例**
  - 竞品/替代方案矩阵（能力差异 + 迁移门槛）
  - 1 份基础路演讲稿草案（问题→方案→证据→风险→下一步）

### 中级：能推动“范围冻结 + 指标口径 + 验证闭环”

- **需要学习什么（方法）**
  - 需求做减法：MoSCoW（必须/应该/可以/不做）与范围冻结写法
  - 指标体系：北极星指标、漏斗指标、样本偏差与限制说明
  - 增长/验证实验：假设→动作→指标→结论→下一步
- **需要掌握什么（能力标准）**
  - 能把需求砍成 **MVP 可交付闭环**，并写清楚“为什么砍、砍到哪、风险是什么”
  - 能定义 **1 个主指标 + 1 个辅指标**（可解释、可测、可验收）
  - 能把用户反馈整理成“可复现问题清单”（现象、触发条件、影响范围）
- **你应该能交付什么（稳定交付包）**
  - 冻结后的 MVP 范围文档（含变更规则）
  - 增长/验证实验日志（至少 1 次）+ 结论
  - 《商业验证报告》（偏差与限制要写清楚）
  - UAT 验收单（对照指标与范围，逐条判定通过/不通过）

### 高级：能带队“方向取舍 + 风险体系 + 叙事防御”

- **需要学习什么（方法）**
  - 方向取舍：决策矩阵（收益/成本/风险/可交付性）与备选方案
  - 护城河/壁垒分析：差异化如何成立、如何被验证、如何被质疑
  - Q&A 防御：把常见质疑预演成“证据化回答”
- **需要掌握什么（能力标准）**
  - 能做出明确取舍：做 A 不做 B 的理由能抗质疑
  - 能建立风险登记：合规/隐私/幻觉/数据可信度，并给出降级策略与验收条款
  - 能把“证据→价值→差异化”固化为团队统一叙事（避免口径不一致）
- **你应该能交付什么（带队交付包）**
  - 方向取舍表 + 风险登记（含降级策略）
  - 结构化 Pitch Deck（5 分钟版）+ Q&A 清单 + 防御演练结论
  - 行业洞察/白皮书（可选加强）+ 商业化下一步路线（1–2 周/1–2 月）

## 3. 互锁协作：你必须卡住的三件事

- **与产品（Product）**：没有证据就不让原型扩张；需求变更必须同步“证据与指标”口径
- **与技术（Tech）**：不承诺做不到的事；对幻觉/隐私/合规风险提出“验收条款”
- **对团队（All）**：用 DoD 验收交付，而不是“我觉得差不多了”

## 4. 技能树（Market Skill Tree）

- **洞察与研究**
  - 用户画像（ICP）与场景拆解
  - 访谈：招募、提纲、追问、记录、引用摘录
  - 竞品/替代方案：能力矩阵、差异点、成本与迁移门槛
  - JTBD / 5W1H / 5 Why（把“情绪”翻译成“行为与约束”）
- **价值评估与指标**
  - UVP（一句话价值主张）与范围冻结（MoSCoW）
  - 北极星指标与漏斗（曝光→激活→留存→转化）
  - UAT 验收口径（指标、样本、偏差、通过/不通过条件）
- **验证与增长**
  - 实验日志：假设→动作→指标→结论→下一步
  - 冷启动：种子用户、渠道、内容与社群打法
  - 商业验证报告（限制与偏差说明）
- **叙事与交付**
  - Pitch Deck（5 分钟结构）与 Q&A 防御
  - 行业洞察/白皮书（可选加强）
  - 简历叙事（STAR/数据化职责与结果）
